I'm pretty pants at Sudoku for someone who spent the majority of their academic life studying Maths - so I wrote a cheat code.

It's a pretty neat little command line solver which runs in either "solve" or "check" mode and helped cement the basics of programming in C. It prints either a full or partial solution to screen/file depending on command line arguments and whether or not a solve is possible. It implements a back-tracking algorithm and was it was real interesting to have a play with.

I'd quite like to take this one a little further in the near future by way of a GUI, or probably in a Web App, to make it a little more user friendly - watch this space.

Having read through T.H.Cormen and Co.'s 'Introduction to Algorithms', I wanted to create a web app that would help to visualise what was going on 'under the hood'.

I wanted to approach this project with a slightly more advanced methodology and decided to use the React.js library coupled with Redux. It was a steep learning curve but well worth the reward as the final product (to be deployed shortly) was lightning fast.

Of course the first algorithm I chose to use was Bubble-Sort. It was nice to get the animations moving and working without having to worry about much else bar swapping elements. I went on to implement visualisations of Heap-Sort, Merge-Sort, Radix-Sort and Quick-Sort also. It was certainly easier to implement the "in-place" algorithms without having to worry about conacatentation and animation of various auxilliary arrays.

All in, a solid project with a steep learning curve but a happy end result!
